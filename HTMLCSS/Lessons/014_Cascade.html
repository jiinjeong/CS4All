The Cascade

What happens if two or more of our CSS rules contradict each other and have equal specificity? The latter (bottom-most) rule will be applied.

The computer reads a CSS file from top to bottom. In the case of equivalent rules, the rule that is lower-down will overwrite the rule that is higher-up.

When loading this webpage, the computer first applied a color of red to this blockquote. But then, it overwrote that rule and applied a color of blue to this blockquote.

What if we have two conflicting rules that aren't exactly equivalent but do have equal specificity? Again, the rule that is lower-down will be applied.

This blockquote has two classes: "first-class" and "second-class". A color of green is applied to the class "first-class", and a color of purple is applied to the class "second-class". Since the rule targeting "second-class" is beneath the other in the CSS code, this text will be purple. But if you cut that rule and paste it at the top of the CSS document, this text will turn green.

All this is called the cascade.


However, the cascade is not the deciding factor when an element has inherited a rule that contradicts a rule that it has inherited from a differnt parent.

In this case, CSS applies the rule from the parent that is closer to element in the HTML code. In other words, CSS applies the rule from the parent in which the element is nested fewer layers deep.

For example:

This is a paragraph nested inside of a "div" tag. It will inherit the color applied to the "div" tag: fuschia.

  • This list item is nested two layers deep in the "div" tag, but only one layer deep in the "ul" tag. Therefore, it will display the color applied to the "ul" tag: orange.
